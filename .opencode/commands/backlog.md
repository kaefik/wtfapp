Тебе нужно работать с файлом BACKLOG.md проекта.

**Действие:** $1
**Аргумент:** $2

## Режимы работы

### Режим добавления (idea/bug/test)

Если $1 = idea/идея, bug/баг или test/тест:

1. Прочитай текущее содержимое файла @BACKLOG.md
2. Определи категорию:
   - idea/идея → секция "## Идеи и возможности"
   - bug/баг → секция "## Баги"
   - test/тест → секция "## Запланированные тесты" (если нет — создать после "## Баги")
3. Добавь запись ($2) в конец соответствующей секции в формате:
   - Идеи: `- **Название**: Описание`
   - Баги: `- **Описание бага**: Шаги воспроизведения / контекст`
   - Тесты: `- **Что тестируем**: Описание проверяемого сценария`
4. НЕ изменяй существующие записи
5. Сохрани файл

### Режим выполнения (done)

Если $1 = done/выполнено:

1. Прочитай текущее содержимое файла @BACKLOG.md
2. Найди строку, содержащую текст из $2, в секциях "## Идеи и возможности" или "## Баги"
   - Ищи по вхождению текста из $2 в название записи (текст между `** **`)
   - Если точного совпадения нет — ищи частичное совпадение
3. Вырежи найденную строку целиком (одна строка, начинается с `- **`)
4. Вставь её в конец секции "## Выполненные идеи"
5. НЕ изменяй текст самой записи
6. Сохрани файл
7. Выведи что именно было перемещено

Если запись не найдена — выведи список всех доступных записей и попроси уточнить.

Примеры использования:
- /backlog idea "Автоочистка корзины" "Удалять задачи из корзины через 30 дней"
- /backlog bug "Фильтры не сбрасываются" "При переходе на главную фильтры применяются"
- /backlog test "Экспорт задач" "Проверить что JSON экспорт содержит все данные"
- /backlog done "Изменение пароля"
- /backlog done "Светофор"
- /backlog done "Устаревшие зависимости"
